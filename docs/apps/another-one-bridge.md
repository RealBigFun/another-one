# `another-one-bridge/` ŌĆö Rust bridge crate for Flutter clients

Wraps everything the Dart side needs ŌĆö `LocalSession` for
in-process FFI, `IrohSession` for remote-daemon transport,
the embedded daemon's boot + PTY drain, pair-mobile QR ŌĆö behind
a narrow API the Flutter desktop ([[another-one]]) consumes
through `flutter_rust_bridge`.

The original `mobile-core` name was retired during the
GPUI ŌåÆ Flutter migration. After Phase 6 the desktop is the
crate's primary client; mobile pairs with the bridge's
embedded daemon over iroh.

## Entry points

- `another-one-bridge/src/lib.rs` ŌĆö re-exports + module wiring.
- `another-one-bridge/src/api/local_session.rs` ŌĆö the
  `LocalSession` FRB surface. Reads + writes the shared
  `RegistryState` directly (no network round-trip). All the
  desktop's data verbs live here:
  projects (list / add / remove / rename), tasks (create /
  delete / pin / review-task / new-task), git (changed files,
  commits, checks, compare, branch settings, toolbar actions),
  terminal tabs (attach / detach / launch / activate / close /
  pin / send bytes / resize), custom actions (list / save /
  delete / run), settings (agents / open-in / git scripts /
  shortcuts / mcp), open-in apps, GitHub URLs, build info,
  resource usage, pair-mobile, ŌĆ”
- `another-one-bridge/src/api/iroh_client.rs` ŌĆö `IrohSession`,
  used by mobile peers + remote-host UIs.
- `another-one-bridge/src/api/embedded_daemon.rs` ŌĆö
  `bootEmbeddedDaemon()` from Dart spins up the in-process
  iroh endpoint and the [[#PTY drain]].
- `another-one-bridge/src/embedded_daemon.rs` ŌĆö non-FRB boot
  internals: load `ProjectStore`, build the registry, register
  with the bridge's local-pair info store, spawn the daemon
  thread (`run_endpoint`).
- `another-one-bridge/src/pty_drain.rs` ŌĆö the PTY drain (see
  below).
- `another-one-bridge/src/frb_generated.rs` ŌĆö auto-generated by
  `flutter_rust_bridge_codegen generate`. Never hand-edit.

## Regenerating Dart bindings

```sh
cd another-one
flutter_rust_bridge_codegen generate
cargo build -p another_one_bridge
```

Bindings land in `another-one/lib/src/rust/`.

## Cross-compile

cargokit (vendored under `another-one/rust_builder/`) calls

## PTY drain

The desktop binary used to drive PTY launches from GPUI's
render-tick: every frame it drained
`RegistryState::pending_tab_launches`, called
`spawn_terminal_launch`, and on `Launched` published the
broadcast + writer back onto the registry.

The Flutter desktop has no GPUI tick. Phase 6 of the migration
ported the slimmed-down equivalent into a dedicated
`another-one-pty-drain` OS thread booted from
`embedded_daemon::boot`. It runs at 50ms tick:

1. `mem::take` `pending_tab_launches`. For each request:
   resolve cwd + launch config + agent launch args from the
   registry, mark `in_flight_launches`, call
   `spawn_terminal_launch`.
2. Drain the reply mpsc:
   - `Launched` ŌåÆ publish `output_broadcast` +
     `Arc<Mutex<writer>>` on the registry; drain
     `pending_post_launch_input` (custom-action shell runs).
     Hold the runtime's `master` + `child_killer` locally so
     the child shell doesn't get SIGHUP.
   - `Output` ŌåÆ no-op (the broadcast already routes to
     subscribers via `attach_tab`'s forwarder loop).
   - `Exited` / `Failed` ŌåÆ drop the live entry + clear the
     registry's broadcasts / writers.

What's intentionally not in this drain (vs GPUI's tick):
no VT parsing, no snapshot rebuilds ŌĆö `xterm.dart` consumes
raw bytes directly. Toast surfacing for `Failed` is a
follow-up.

## Design notes

### Dedicated tokio runtime
FRB's default async executor is a thread-pool, not a tokio
runtime. iroh's UDP sockets and internal actor tasks require
tokio specifically, so the bridge spins up a static
`OnceLock<Runtime>` at `init_app()` time. See
[[../architecture/frb-tokio-runtime]] and
[[../postmortems/2026-04-23-iroh-android-hang]].

### No address lookup (mobile path)
The mobile peer binds with `presets::Minimal` (no DNS) and
`RelayMode::Disabled`, takes direct socket addrs in the connect
call. Avoids Android's broken `/etc/resolv.conf` path and
keeps the sandbox free of dependencies on the public iroh
relay mesh.

### Opaque handle pattern
`LocalSession` and `IrohSession` are both `#[frb(opaque)]` ŌĆö
Dart sees them as opaque types with async methods, but the
struct + tokio tasks live entirely in Rust. Teardown
coordinates via `Mutex<Option<oneshot::Sender>>`: `close()`
signals, background tasks see the signal, close the
underlying connection cleanly.

## Known gaps

- **Patched `noq-udp`** ŌĆö `vendor/noq-udp/` is a vendored fork

  Remove once upstream ships a fix. See
