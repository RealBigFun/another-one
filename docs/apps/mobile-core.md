# `mobile-core/` — Rust core for mobile clients

Wraps Iroh client operations (connect, bidirectional byte stream,
close) behind a narrow API that [[mobile]] consumes through
`flutter_rust_bridge`. This is the crate that eventually gets absorbed
into the broader `core` library that [[desktop]] will also consume —
see [[../architecture/peer-to-peer-nodes]].

## Entry points

- `mobile-core/src/lib.rs` — re-exports.
- `mobile-core/src/api/iroh_client.rs` — the FRB-annotated API:
  - `iroh_connect(endpoint_id, direct_addrs) -> IrohSession`
  - `IrohSession::send(bytes)` / `subscribe(sink)` / `close()`
  - `init_app()` installs tracing subscriber + spins up the dedicated
    tokio runtime (see [[../architecture/frb-tokio-runtime]]).
- `mobile-core/src/frb_generated.rs` — auto-generated by
  `flutter_rust_bridge_codegen generate`. Never hand-edit.

## Regenerating Dart bindings

```sh
cd mobile
flutter_rust_bridge_codegen generate
cargo build -p mobile_core
```

Bindings land in `mobile/lib/src/rust/`.

## Cross-compile

cargokit (vendored under `mobile/rust_builder/`) calls
`rustup run stable cargo build --target <android-triple>` for each
Android ABI during `flutter build`. See [[mobile]] for environment
prereqs.

## Design notes

### Dedicated tokio runtime
FRB's default async executor is a thread-pool, not a tokio runtime.
iroh's UDP sockets and internal actor tasks require tokio specifically,
so `iroh_connect` delegates onto a static `OnceLock<Runtime>` spun up
at `init_app()` time. See [[../architecture/frb-tokio-runtime]] and
[[../postmortems/2026-04-23-iroh-android-hang]].

### No address lookup
The client binds with `presets::Minimal` (no DNS address lookup) and
`RelayMode::Disabled`, and takes direct socket addrs in the connect
call. Avoids Android's broken `/etc/resolv.conf` path and keeps the
sandbox free of dependencies on the public iroh relay mesh.

### Opaque handle pattern
`IrohSession` is `#[frb(opaque)]` — Dart sees it as `IrohSession`
with async methods, but the actual struct + tokio tasks live entirely
in Rust. Teardown coordinates via a `Mutex<Option<oneshot::Sender>>`:
`close()` signals, background tasks see the signal, close the
underlying connection cleanly.

## Known gaps

- No send-side framing yet — `IrohSession::send` writes raw bytes. Will
  add length-prefixed frames to support resize/control messages on the
  Iroh path (matching [[daemon-sandbox]]'s WS control protocol).
- iroh crate is pre-1.0 (0.98.x); expect minor-version churn.
